# 주말 독서 토론 참가 관리 서비스 — 전체 흐름도

이 문서는 `index.html`(고객 페이지)과 `admin.html`(관리자 페이지) 두 파일의
함수 단위 호출 관계와, Supabase `reservations` 테이블을 통한 데이터 이동을 정리한 것입니다.

- 두 파일은 서로 직접 호출하지 않고, 같은 Supabase 프로젝트의 `reservations` 테이블을 통해서만 연결됩니다.
- `index.html`은 anon(비로그인) 권한으로 INSERT만 가능합니다.
- `admin.html`은 로그인(authenticated) 후에만 SELECT/UPDATE/DELETE가 가능합니다.

---

## 1. 전체 아키텍처 (파일 ↔ Supabase 데이터 흐름)

```mermaid
flowchart LR
    subgraph CustomerPage["index.html (고객 페이지)"]
        CustomerSubmit["reservationForm submit 이벤트"]
    end

    subgraph AdminPage["admin.html (관리자 페이지)"]
        AdminLogin["로그인 폼 submit"]
        AdminLoad["접수내역불러오기()"]
        AdminConfirm["예약확정처리()"]
        AdminDelete["예약삭제처리()"]
        AdminLogout["로그아웃 버튼 클릭"]
    end

    AuthSvc[["Supabase Auth<br/>(이메일/비밀번호)"]]
    Table[("reservations 테이블<br/>name, contact, hope_date,<br/>request, status, created_at")]

    CustomerSubmit -- "INSERT<br/>(anon_insert 정책)" --> Table

    AdminLogin -- "signInWithPassword" --> AuthSvc
    AuthSvc -- "세션 발급" --> AdminLoad
    AdminLogout -- "signOut" --> AuthSvc

    AdminLoad -- "SELECT<br/>(authenticated_select 정책)<br/>order by created_at desc" --> Table
    Table -- "행 데이터 반환" --> AdminLoad

    AdminConfirm -- "UPDATE status='확정'<br/>(authenticated_update 정책)" --> Table
    AdminDelete -- "DELETE<br/>(authenticated_delete 정책)" --> Table
```

## 2. index.html — 고객 예약 문의 흐름

```mermaid
flowchart TD
    Load(["페이지 로드"]) --> Bind["reservationForm에<br/>submit 이벤트 리스너 등록"]
    Bind --> Wait(["사용자가 '예약 문의하기' 버튼 클릭"])

    Wait --> Validate{"필수 입력 검사<br/>(이름 / 연락처 / 희망 날짜)"}
    Validate -- "빈 값 있음" --> ShowError["formMessage에 에러 문구 표시<br/>해당 입력칸 focus"]
    ShowError --> Wait

    Validate -- "모두 입력됨" --> Lock["submitButton.disabled = true<br/>(중복 제출 방지)"]
    Lock --> Insert["supabaseClient
        .from('reservations')
        .insert(name, contact, hope_date, request)"]
    Insert --> DB[("reservations 테이블")]
    Insert --> Unlock["submitButton.disabled = false"]

    Unlock --> Result{"error 발생?"}
    Result -- "예" --> FailMsg["formMessage에<br/>'접수 중 문제 발생' 표시"]
    Result -- "아니오" --> OkMsg["formMessage에<br/>'접수되었습니다' 표시<br/>reservationForm.reset()"]
```

## 3. admin.html — 관리자 로그인 · 접수 관리 흐름

```mermaid
flowchart TD
    Start(["스크립트 로드"]) --> Init["로그인상태확인()"]
    Init --> GetSession["supabaseClient.auth.getSession()"]
    GetSession --> HasSession{"세션 있음?"}
    HasSession -- "없음" --> ShowLogin["로그인화면보이기()<br/>로그인 폼만 노출"]
    HasSession -- "있음" --> ShowAdmin["관리자화면보이기()"]

    %% 로그인
    LoginSubmit(["로그인 폼 submit"]) --> ValidateLogin{"이메일/비밀번호<br/>입력 확인"}
    ValidateLogin -- "비어있음" --> LoginErr1["loginMessage 에러 표시"]
    ValidateLogin -- "입력됨" --> SignIn["supabaseClient.auth
        .signInWithPassword(email, password)"]
    SignIn --> AuthSvc[["Supabase Auth"]]
    SignIn --> SignInResult{"error 발생?"}
    SignInResult -- "예" --> LoginErr2["loginMessage에<br/>'이메일 또는 비밀번호가<br/>올바르지 않습니다' 표시"]
    SignInResult -- "아니오" --> ShowAdmin

    %% 로그아웃
    LogoutClick(["로그아웃 버튼 클릭"]) --> SignOut["supabaseClient.auth.signOut()"]
    SignOut --> AuthSvc
    SignOut --> ShowLogin

    %% 접수 내역 조회
    ShowAdmin --> LoadList["접수내역불러오기()"]
    LoadList --> SelectQuery["supabaseClient
        .from('reservations').select(...)
        .order('created_at', desc)"]
    SelectQuery --> DB[("reservations 테이블")]
    SelectQuery --> SelectResult{"error 발생?"}
    SelectResult -- "예" --> AdminErr["adminMessage 에러 표시"]
    SelectResult -- "아니오" --> RenderTable["표그리기(data)"]

    RenderTable --> FormatTime["접수시각포맷(created_at)<br/>행마다 호출"]
    RenderTable --> MakeConfirmBtn["행마다 [확정] 버튼 생성<br/>클릭 시 예약확정처리(id) 바인딩"]
    RenderTable --> MakeDeleteBtn["행마다 [삭제] 버튼 생성<br/>클릭 시 예약삭제처리(id) 바인딩"]
    RenderTable --> Badge["상태값에 따라<br/>status-badge--대기(회색) /<br/>status-badge--확정(초록) 클래스 부여"]

    %% 확정 처리
    MakeConfirmBtn -.클릭.-> ConfirmFn["예약확정처리(id, button)"]
    ConfirmFn --> UpdateQuery["supabaseClient
        .from('reservations')
        .update(status: '확정')
        .eq('id', id)"]
    UpdateQuery --> DB
    UpdateQuery --> ConfirmResult{"error 발생?"}
    ConfirmResult -- "예" --> AdminErr
    ConfirmResult -- "아니오" --> LoadList

    %% 삭제 처리
    MakeDeleteBtn -.클릭.-> DeleteFn["예약삭제처리(id, button)"]
    DeleteFn --> ConfirmDialog{"confirm() 확인창<br/>'정말 삭제하시겠습니까?'"}
    ConfirmDialog -- "취소" --> End1(["아무 동작 없이 종료"])
    ConfirmDialog -- "확인" --> DeleteQuery["supabaseClient
        .from('reservations')
        .delete()
        .eq('id', id)"]
    DeleteQuery --> DB
    DeleteQuery --> DeleteResult{"error 발생?"}
    DeleteResult -- "예" --> AdminErr
    DeleteResult -- "아니오" --> LoadList
```

## 4. 함수 목록 요약

### index.html
| 함수/핸들러 | 호출하는 대상 | 데이터 이동 |
|---|---|---|
| `reservationForm` submit 리스너 (익명 함수) | `supabaseClient.from('reservations').insert()` | 폼 입력값 → `reservations` 테이블 (INSERT) |

### admin.html
| 함수 | 호출하는 대상 | 호출되는 시점 |
|---|---|---|
| `로그인상태확인()` | `supabaseClient.auth.getSession()`, `관리자화면보이기()`, `로그인화면보이기()` | 스크립트 최초 로드 시 |
| `로그인화면보이기()` | (DOM 조작만) | 세션 없음 / 로그아웃 시 |
| `관리자화면보이기()` | `접수내역불러오기()` | 세션 있음 / 로그인 성공 시 |
| `loginForm` submit 리스너 | `supabaseClient.auth.signInWithPassword()`, `관리자화면보이기()` | 로그인 버튼 클릭 시 |
| `logoutButton` click 리스너 | `supabaseClient.auth.signOut()`, `로그인화면보이기()` | 로그아웃 버튼 클릭 시 |
| `접수내역불러오기()` | `supabaseClient.from('reservations').select().order()`, `표그리기()` | 관리자 화면 진입 시 / 확정·삭제 후 |
| `표그리기(reservations)` | `접수시각포맷()`, 각 행의 `예약확정처리()` / `예약삭제처리()` 바인딩 | 접수 내역 조회 성공 시 |
| `접수시각포맷(isoString)` | (순수 변환 함수, Date → 문자열) | `표그리기()`에서 행마다 호출 |
| `예약확정처리(id, button)` | `supabaseClient.from('reservations').update().eq()`, `접수내역불러오기()` | [확정] 버튼 클릭 시 |
| `예약삭제처리(id, button)` | `confirm()`, `supabaseClient.from('reservations').delete().eq()`, `접수내역불러오기()` | [삭제] 버튼 클릭 시 |
